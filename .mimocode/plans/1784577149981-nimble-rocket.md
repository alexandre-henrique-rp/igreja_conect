# Plano: Senha reforçada, Rate Limit 3/1h, Fluxo de Convite

## Resumo

4 funcionalidades:
1. Validação de senha: 1 maiúscula + 1 número + 1 especial (backend)
2. Rate limit: 3 tentativas → bloqueio 1h, desbloqueio via admin na página do membro
3. Fluxo de convite: admin cadastra com cargo → modal com texto copiável (WhatsApp/Telegram) → membro cria senha
4. Sidebar: esconder Configurações

---

## 1. Validação de senha reforçada

### `app/lib/validators/auth.ts`
- Schema `LoginInputSchema.senha`: adicionar regex `^(?=.*[A-Z])(?=.*\d)(?=.*[!@#$%^&*]).{8,128}$`
- Mensagem: "A senha deve ter no mínimo 8 caracteres, incluindo 1 maiúscula, 1 número e 1 caractere especial."

### `app/lib/schemas/auth.ts`
- Adicionar `SenhaSchema` (validação reforçada para criação de senha via convite)
- Adicionar `ConfirmarSenhaSchema` (senha + confirmação devem ser iguais)

## 2. Rate Limit: 3 tentativas, bloqueio 1h

### `app/lib/rate-limit.server.ts`
- `MAX_FAILS`: 5 → **3**
- `WINDOW_MS`: 15 min → **3600000** (1h)
- Nova função `unblockIP(ip: string)`: remove o bucket do Map
- Nova função `getBlockedIPs()`: retorna Array<{ ip, count, firstAt, retryAfter }>

### `app/routes/public/login.tsx`
- Mensagem de rate limit: "Muitas tentativas. Tente novamente em 1 hora."

### `app/routes/app/membros.$id.tsx`
- No loader: incluir status de bloqueio do IP (via `getBlockedIPs`)
- No action: novo `intent === "unblock"` — chama `unblockIP`, retorna redirect

  - Botão "Desbloquear acesso" (só ADMIN, só se bloqueado)
  - Histórico de tentativas失败 (últimas 10, via safeLog parsing)

## 3. Fluxo de Convite

### 3.1 Model

**`prisma/schema.prisma`**
```prisma
model ConviteToken {
  id        String   @id @default(uuid())
  membroId  String
  token     String   @unique
  expiresAt DateTime
  usedAt    DateTime?
  createdAt DateTime @default(now())

  membro Membro @relation(fields: [membroId], references: [id], onDelete: Cascade)

  @@index([token])
  @@map("convite_tokens")
}
```

No model `Membro`, adicionar:
```prisma
conviteTokens ConviteToken[]
```

### 3.2 Service

**`app/lib/convite.server.ts`** (novo)
- `criarConvite(membroId, membroNome, cargo): Promise<{ token, url, textoConvite }>`
  - Gera UUID, insere `ConviteToken` (expiresAt = now + 2h)
  - Monta texto em markdown para WhatsApp/Telegram:
    ```
    Olá {nome}! 👋

    Você foi convidado(a) para fazer parte do sistema **Igreja Connect** como *{cargo formatado}*.

    Para criar sua senha e acessar o sistema, clique no link abaixo (válido por 2 horas):

    {url}

    Se tiver dúvidas, entre em contato com a administração da igreja.
    ```
- `validarConvite(token): Promise<{ membroId, membroNome } | null>`
- `usarConvite(token): Promise<void>`

### 3.3 Rota pública

**`app/routes/public/convite.$token.tsx`** (novo)
- **Loader:** valida token → se inválido/expirado → 404
- **Action:** valida senha + confirmação → hash → atualiza `Membro.senhaHash` → marca convite usado → redirect `/login?motivo=senha-criada`
- **UI:** form dark (estilo login) com nome do membro, senha, confirmação, checklist visual de requisitos

### 3.4 FormMembro: campo Cargo

**`app/components/FormMembro.tsx`**
- Adicionar Select `cargo` na seção "Vinculação"
- Opções: (sem acesso), ADMIN, PASTOR, SECRETARIO, DISCIPULADOR, FINANCEIRO, LIDER_MINISTERIO
- Só exibir para ADMIN

**`app/lib/schemas/membros.ts`**
- Adicionar `cargo` opcional ao `MembroCreateSchema`

**`app/lib/members.server.ts`**
- `createMembro`: incluir `cargo` no data se fornecido

### 3.5 Modal de convite

**`app/routes/app/membros.novo.tsx`**
- Action: se membro criado tem email E cargo → chamar `criarConvite` → retornar `{ convite: { url, textoConvite, membroNome } }`
- Componente: quando `actionData.convite` existe, mostra modal com:
  - Texto formatado (bloco de código pre-formatado)
  - Botão "Copiar texto" (usa clipboard API)
  - Link direto do convite (copiável)
  - Botão "Fechar"

### 3.6 Sidebar

**`app/components/Sidebar.tsx`**
- Adicionar propriedade `hidden?: boolean` ao tipo `MenuItem`
- Item "Configurações": `hidden: true`
- Filtrar: `MENU_ITEMS.filter(item => !item.hidden)` no render
