1. Objetivo

Esta especificação é a fonte de verdade para implementação do ciclo 4 (PRD). Cobre as 5 páginas de leitura agregada em /app/financeiro/relatorios/** (Hub + DRE + Balancete + Fluxo de Caixa + Customizado) e a exportação CSV — tudo read-only sobre o Módulo Financeiro do ciclo 2. Sem migration, sem novos models Prisma, sem novas regras de negócio.

Escopo do ciclo 4: 5 services de agregação em app/lib/relatorios.server.ts + 1 service CSV em app/lib/relatorios-csv.server.ts + 5 rotas em app/routes/app/financeiro/relatorios/** + 2 componentes compartilhados (<FiltrosPeriodo />, <KpiCard />) + ~25 ícones SVG inline em app/components/icons/FinanceIcons.tsx + item "Relatórios" no Sidebar.tsx.

Decisões herdadas dos ciclos 1–3 (imutáveis): monólito modular React Router 7, Prisma 7 + SQLite, Tailwind 4, Vite 8, TypeScript 5.9 strict, Zod 4, bcryptjs. Decisões específicas do ciclo 4: CSV (não PDF, decisão §5.2), SVG inline (não lib externa, decisão §5.5), placeholders visuais para "Projeção" e "Status" (decisões §5.6 e §5.7).

4.2 Decisões arquiteturais (referência)

  • ADR-001 a ADR-007 herdadas dos ciclos 1–3 (sessão, bcryptjs, Zod, monolito, singleton Prisma, transferência híbrida, trava de saldo no service).
  • ADR-008 (ciclo 4): 5 services de relatório read-only — sem $transaction de escrita, sem mutação. Ordem canônica: assertCanSeeRelatorios → assertPeriodoValido → groupBy/findMany.
  • ADR-009 (ciclo 4): soma sempre em Int cents, formatação BRL só na borda UI (estende convention-monetary-values).
  • ADR-010 (ciclo 4): filtros de data sempre semi-abertos [gte, lt) (estende pattern-relatorios-aggregations).
  • ADR-011 (ciclo 4): separação de responsabilidade: relatorios-csv.server.ts é separado de relatorios.server.ts (formato de export ≠ regra de negócio). Helpers reutilizáveis em futuros exports (PDF, JSON).
  • ADR-012 (ciclo 4): export CSV via action (não loader), com Cache-Control: no-store (LGPD).

5.7 Detecção de prompt injection

Os 5 endpoints tratam todos os inputs (query params) como DADOS. Mesmo padrão do ciclo 1. Zod .strict() rejeita campos extras. Descrições textuais em CSV passam por escapeCsvField (RFC 4180) que previne CSV injection.

7.1 app/lib/relatorios.server.ts (5 services read-only)

7.1.1 getDRE(periodo, user)

/**
 * @description Agrega entradas e saidas por categoria em 1 periodo (DRE).
 * @param {{ inicio: Date; fim: Date }} periodo - Range semi-aberto [inicio, fim).
 * @param {SessionUser} user - Usuario autenticado.
 * @returns {Promise<DREData>} DTO com totais, resultado liquido e listas por categoria.
 * @throws {Response} 403 se user sem cargo em RELATORIOS_CARGOS.
 * @throws {Response} 400 se periodo.inicio >= periodo.fim.
 */
export async function getDRE(periodo, user): Promise<DREData> {
  assertCanSeeRelatorios(user);
  assertPeriodoValido(periodo);

  const [entradas, saidas] = await Promise.all([
    prisma.lancamento.groupBy({
      by: ["categoria"],
      where: { dataCompetencia: { gte: periodo.inicio, lt: periodo.fim }, tipo: "ENTRADA" },
      _sum: { valorCentavos: true }, _count: { _all: true }
    }),
    prisma.lancamento.groupBy({
      by: ["categoria"],
      where: { dataCompetencia: { gte: periodo.inicio, lt: periodo.fim }, tipo: "SAIDA" },
      _sum: { valorCentavos: true }, _count: { _all: true }
    })
  ]);

  const totalEntradasCentavos = entradas.reduce((acc, e) => acc + (e._sum.valorCentavos ?? 0), 0);
  const totalSaidasCentavos = saidas.reduce((acc, s) => acc + (s._sum.valorCentavos ?? 0), 0);

  return {
    periodo,
    totalEntradasCentavos, totalSaidasCentavos,
    resultadoLiquidoCentavos: totalEntradasCentavos - totalSaidasCentavos,
    entradasPorCategoria: mapCategorias(entradas, totalEntradasCentavos),
    saidasPorCategoria: mapCategorias(saidas, totalSaidasCentavos)
  };
}

8.3 Defesa contra Prompt Injection

O ciclo 4 herda do ciclo 1 a regra: todos os inputs são DADOS, nunca instruções. Mitigações específicas:

  • Query params: Zod .strict() rejeita campos extras. Loader lança 400 em payload inválido.
  • Descrições textuais em CSV: passam por escapeCsvField (RFC 4180 §2.6). Aspas internas viram "". Vírgula, ponto-e-vírgula e quebra de linha forçam aspas ao redor do campo.
  • Logs de auditoria: safeLog com allowlist. Proibido em log: valorCentavos, descricao, membro.nome, caixa.nome, conteúdo de query params.
  • UI rendering: React escapa por padrão. Sem dangerouslySetInnerHTML.

11. Performance e limites

11.1 Volumetria esperada

  • Igreja típica: ~5.000 lançamentos/mês (média). Pico de fechamento mensal: ~500 lançamentos em 1 dia.
